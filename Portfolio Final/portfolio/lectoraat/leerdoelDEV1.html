DEV

"De student heeft een applicatie gerealiseerd met een module pattern. Externe libraries zijn gebruikt om de applicatie te ontwerpen. De applicatie is tot stand gekomen via user stories en een schets van de architectuur."

Voor mijn concept heb ik eerst gekeken naar hoe we de applicatie maken. Dit is gedaan met behulp van een architectuur diagram. Deze heeft meerdere iteraties omdat ik langzaam heb geleerd wat erin moest. Voor mijn eerste feedback moest ik het meer gedetailleerd maken, aangezien alleen de app in het algemeen liet zien. Mijn tweede iteratie was een stuk gedetailleerder, hoewel deze opgeschoond kon worden en nog niet liet weten wat met elkaar communiceren en hoe. Dit is hoe ik op de laatste iteratie ben uitgekomen.

Nadat ik de architectuur diagram had als basis om te kijken welke functionaliteiten in mijn applicatie moest zitten, ben ik begonnen met programmeren.

Het eerste dat ik heb gemaakt was een punten convertor. Om mijzelf modern JS aan te leren wilde ik hiermee de functie schrijven van de punten convertor. Na wat onderzoek te hebben gedaan hoe de basisdingen moeten van moderne Javascript heb ik de functie geschreven. Daarbij kreeg ik feedback hoe het nog netter wordt en als tip dat andere mensen het ook moeten gebruiken. Deze puntenconvertor is meer een demo van hoe het later behind de scenes zou moeten werken, maar voor nu is het een website met een simpele UI. De fitcoins die uit worden gerekend worden naar de database gestuurd, net zoals het werd gezegd in het architectuurdiagram.

Ik heb ook met ReactJS een login scherm gemaakt. Ik wilde met ReactJS iets gaan doen, aangezien huidige bedrijven veel met React werken en mij het wel handig leek om het dus ook te gaan leren. Ik begon met een tutorial om de basics te leren, na mijn eerste simpele checkbox gemaakt te hebben wat dingen opslaat snapte ik niet waarom React wordt gebruik. Naar mij ervaring is het onoverzichtelijk en te ingewikkeld. Maar toen ik de basis snapte en begon aan de login scherm werd het duidelijk waarom het gebruikt wordt bij zoveel bedrijven. Het kan dezelfde front-end doorgeven aan IOS en Android, wat superhandig is aangezien je dan niet dingen dubbel hoef te maken. Ook hoeven geen 100 javascript bestanden gekoppeld te worden aan 1 html, maar zit elke functie of andere dingen in aparte JS-bestanden. Ik heb nu afgelopen 2 weken de basis van ReactJS geleerd, en hoop later met meer programmeer challenges meer met React kan werken en dus daardoor beter erin ga worden. Hiernaast kun je de login scherm vinden, die nog niet geconnect is naar de Database, waar je met element-inspect kunt lezen wat ik heb gedaan. Kort wat ik aan code heb geleerd, dingen moeten in aparte javascript bestand, manier van hoe dingen moeten worden geschreven worden in React en waarom dit hoort.
